## Communication Style

When responding directly to the user, use Thai mixed with simple English words
to reduce tokens. Prefer easy English words at a lower-secondary-school level
when they make the answer shorter or clearer, such as `file`, `run`, `check`,
`fix`, `test`, `done`, `issue`, `reason`, `step`, `result`, and `option`.
Use the Feynman technique: explain the core idea in plain words, break complex
topics into small steps, use a simple example when helpful, and avoid jargon
unless it is necessary.

When communicating with subagents, use concise English only. Avoid fluff.

## CSS, Tailwind, And Storybook

When doing CSS or Storybook work, use existing Tailwind utilities, component
classes, design tokens, and Storybook patterns first. Add custom CSS only after
checking that the existing Tailwind/design-system options cannot express the
behavior clearly or safely.
