Implementation Plan: Sagittarius Production Web Redesign Fixes

Goal: turn the June 9 redesign audit and issue index into an executable, reviewed plan for fixing cockpit craft issues without resetting the product. The product lane stays Calm Travel Ops: dense, calm, friendly, trustworthy, scan-friendly, and specific to group trip planning.

Changed New Removed Shared Risky

Current vs Proposed Structure

Current

Most important current issue: cockpit surfaces inherit decorative postcard treatment and several mobile workflows rely on dense desktop controls.

Proposed

Most important proposed difference: cockpit chrome becomes operational, while travel warmth moves to photos, map pins, empty states, and success moments.

Critical Risks

Risk Impact Mitigation
Global repaint changes route pages not covered by the audit. Unexpected regressions in account/access or public landing. Scope decorative art to explicit landing/auth classes and test HomeLanding, AccountAccessPanel, and cockpit stories.
Itinerary mobile inspector duplicates mutation logic. Local/API mode behavior or path model can drift. Reuse existing SmartItineraryTable callbacks and data model. Do not create a second itinerary state writer.
Photos cover URLs can be unsafe or broken. Broken cards or privacy-sensitive external loads. Use existing safe URL pattern for links, add thumbnail fallback, and avoid loading unsafe cover URLs.
Map retry can remount MapLibre incorrectly. Memory leaks, duplicate markers, or keyboard focus issues. Keep cleanup tests in RouteMapView.test.tsx, add retry state key, and verify marker cleanup.
Touch target fixes reduce useful desktop density. Organizer editing becomes slower on desktop. Use responsive classes: keep dense desktop rows, raise mobile and frequent controls to 40-44px.
